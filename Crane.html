Segmentation Model for Crane Posture Identification

Scenario: refine tower crane annotations using polygon masks (and shadows), segment the hoisted load, and detect workers in the video. The posture and motion of the crane can then be used to estimate working time and support proximity/risk analytics.

Why it matters

  • Crane utilization / productivity: crane posture and motion provide a reliable signal to infer when the crane is actively working, enabling automatic estimation of effective working time from video.
  • Proximity analytics: polygon segmentation enables computing pixel distances between:
    • worker centroid ↔ hoisted object centroid
    • worker centroid ↔ crane centroid
    These distances can be used as a proxy for near-miss monitoring or for triggering additional checks (e.g., when a worker approaches a suspended load).
  • Better geometry than boxes: masks capture the true shape/orientation of cranes and loads (including shadows), which improves posture estimation and stabilizes downstream measurements.

Method overview (Segment Anything Model)

This module uses Meta’s Segment Anything Model (SAM) to generate object masks for the tower crane, the hoisted object, and workers. SAM is promptable and can produce high-quality masks without task-specific training.

SAM's core innovation lies in its "cued segmentation" task design, which enables the model to generate high-quality segmentation masks based on various user-provided cues (such as points, bounding boxes, masks, and even text), achieving generalization even in zero-shot scenarios.

Key components:

  • Image encoder: converts the input image into a compact feature embedding.
  • Prompt encoder: encodes user prompts, supporting sparse prompts (points / bounding boxes) and dense prompts (masks, text).
  • Mask decoder: a lightweight Transformer module that fuses image embeddings with cue embeddings through self-attention and bidirectional cross-attention, then outputs segmentation masks.

Figure: Prompt-based segmentation for isolating objects (e.g., crane, load, workers) to support posture inference and pixel-distance measurement.

Strengths

  • Strong performance without training: SAM can produce useful masks out-of-the-box, reducing labeling and training costs.
  • Promptable workflow: easy to adapt to different cranes/loads by changing prompts (points/boxes/masks/text) instead of re-training.
  • Geometry-aware outputs: masks capture object posture/orientation and can improve motion/posture identification compared to bounding boxes.
  • Upgrade path: fine-tuning (or training a lightweight head) on site-specific data can further improve mask consistency and accuracy.
  • Enables downstream metrics: once objects are segmented, it becomes straightforward to compute centroids and pixel distances for monitoring.

Limitations

  • Background confusion: accuracy drops when the hoisted object is close to background pixels (low contrast / similar texture).
  • Motion blur sensitivity: segmentation quality degrades significantly under motion blur, which is common in fast-moving loads.
  • Occlusions and clutter: partial occlusion by structures (scaffolds, rebar, columns) can cause mask fragmentation or missed detections.
  • Pixel distance ≠ real distance: without camera calibration or depth estimation, pixel distances are only a proxy and can be misleading under perspective changes.
